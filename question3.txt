หากท่านได้รับมอบหมายให้รับหน้าที่พัฒนาระบบ POS คิดเงินหน้าร้านเกี่ยวกับระบบ Fitness และ มีการประเมินจาก Developer lead หรือ Project Manager 
ว่าต้องใช้เวลาพัฒนาทังหมด 120 วัน และงานในส่วนที่ต้อง Develop กว่า 50 Function ย่อย ใน Project นี้ที่ท่านรับผิดชอบทางทีมไม่ได้กําหนดเวลาเข้าออกงาน 
หรือต้องเข้ามาทํางานที่บริษัท ฯ แต่เน้นผลลัพธ์ของงานเป็นสําคัญ ท่านจะออกแบบวิธีการทํางานของท่านอย่างไร จะใช้วิธีรายงานความคืบหน้างานอย่างไร ทําอย่างไรเมื่อติดปัญหา 
และจะจัดการปัญหาอย่างไรหากเล็งเห็นว่างานเริ่มช้ากว่าปกติ จงอธิบายให้ครบถ้วนแต่ละประเด็น

Question : จะออกแบบวิธีการทํางานอย่างไร
1.	ทำความเข้าใจถึง requirement ของงานร่วมกับทุกคนในทีม เพื่อให้ความเข้าใจในการทำงานของแต่ละ function มีความตรงกันและถูกต้อง
2.	ทำการแบ่งกลุ่มส่วนของ function ย่อยๆ เพื่อแบ่งหน้าที่กันกับคนในทีม เช่น กลุ่ม function ของสมาชิก , กลุ่ม function การจ้าง trainer ใน fitness เป็นต้น
3.	เมื่อได้ส่วนที่รับผิดชอบแล้วก็เริ่มแบ่งเวลาในการทำงานโดยเทียบกับส่วนที่ต้องรับผิดชอบ
4.	เริ่มทำโดยเริ่มจากส่วนหลักที่เป็นงานเบื้องต้นก่อน
5.	ทุกครั้งที่ทำแต่ละ function ย่อยเสร็จให้ทำการ test ดูก่อน แล้วเมื่อรวมเป็นส่วนใหญ่ๆก็ให้ทำการ test อีกครั้ง

Question : จะใช้วิธีการรายงานความคืบหน้างานอย่างไร
อาจจะทำการจัดเป็น Daily scrum meeting หรือ Weekly scrum meeting เพื่อเล่าถึงรายละเอียดงาน หรือปัญหาที่ติดอยู่ เพื่อให้ Developer lead 
หรือ Project Manager หรือ คนในทีมช่วยกันตรวจสอบหรือช่วยแก้ไขปัญหาซึ่งกันและกันและช่วยประเมินในส่วนของการทำงาน รวมถึงพวก logic ของ function ที่จะทำต่อไป

Question : ทําอย่างไรเมื่อติดปัญหา
ทำการหาความรู้หรือศึกษาปัญหาที่ติดผ่านทางอินเตอร์เน็ตก่อน หากไม่สามารถแก้ไขได้ด้วยตัวเองก็จะทำการสอบถามหัวหน้าหรือเพื่อนๆถึงปัญหาดังกล่าว

Question : จะจัดการปัญหาอย่างไรหากเล็งเห็นว่างานเริ่มช้ากว่าปกติ
เร่งการทำงานให้ใช้เวลาให้น้อยลง โดยจะทำการประเมินกับงานที่เหลือกับเวลาที่เหลืออยู่ ซึ่งอาจจะทำงานให้เสร็จก่อนเวลาประมาณ 1 อาทิตย์
